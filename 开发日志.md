### 18. 技术目录 hover 聚焦效果优化

**问题：**
- 原 hover 效果：被 hover 的条目通过 `padding-left: 2rem` 单独向右移动，其他条目不动
- 当 hover 01（第一条目）时，仅 01 右移，02/03/04 不动；hover 02/03/04 时同理
- 用户反馈：hover 01 时效果与 hover 02/03/04 时不一致，希望改为非 hover 条目向左移动

**方案：**
- 移除 `.tech-toc-item:hover` 的 `padding-left: 2rem` 右移效果
- 改为容器级 hover 联动：`.tech-toc:hover .tech-toc-item` 使所有条目向左偏移 6px 并降低透明度至 0.65
- `.tech-toc:hover .tech-toc-item:hover` 将被 hover 条目重置回原位（translateX(0)）并恢复透明度至 1
- 基础 `.tech-toc-item` 新增 `transform: translateX(0)` 确保默认状态无偏移

**效果：**
- hover 任意条目时，该条目保持原位并高亮，其余条目统一向左微移并淡化
- 无论 hover 01、02、03 还是 04，视觉效果完全一致
- `transition: all` 已在基础样式中声明，transform 和 opacity 变化自动获得平滑过渡

`<p>` 是块级元素，默认占父容器 100% 宽度。`white-space: nowrap` 导致文本溢出 `<p>` 的盒子。`text-align: center` 在盒子内居中文本，但溢出部分被 `overflow-x: hidden` 不对称裁剪（左侧被视口边缘裁剪），视觉上表现为左对齐。

学科竞赛页能正常居中是因为文本较短（42字），没有溢出视口。智能抽取页（52字 + 长英文）和苗绣拼图页的文本溢出了视口，导致问题暴露。

**最终修复方案：`width: fit-content`**

所有 `nowrap` 的 `<p>` 元素添加：
```css
width: fit-content;
max-width: 100%;
margin: 0 auto;
```

原理：
1. `width: fit-content` 使 `<p>` 收缩到恰好包裹文本内容的宽度
2. `max-width: 100%` 防止在极窄屏幕上溢出父容器
3. `margin: 0 auto` 将收缩后的 `<p>` 在父容器内水平居中
4. 没有溢出 → 没有裁剪 → 完美居中

应用的元素：
| 元素 | 位置 |
|------|------|
| `.hero-desc` | 首页描述段落 |
| `.detail-hero-desc` | 详情页中文描述（智能抽取、AIGC作品集、苗绣拼图） |
| `.detail-hero-desc-en` | 详情页英文描述 |
| `.award-intro-cn` | 学科竞赛中文描述 |
| `.award-intro-en` | 学科竞赛英文描述 |

**游戏遮罩不消失修复：**

问题：iframe `load` 事件在 `file://` 协议下可能不触发，导致 fallback 遮罩一直显示在游戏上方。

修复：
1. `script.js` 添加 3 秒超时兜底：`setTimeout(hideGameFallback, 3000)`，无论 `load` 事件是否触发，3 秒后自动隐藏遮罩
2. `.game-frame-fallback` 添加 `pointer-events: none`，使遮罩点击穿透，不阻挡游戏交互
3. `.game-open-link` 保持 `pointer-events: auto`，确保链接仍可点击

### 35. 文字居中问题二次修复（text-align: center）、游戏容器溢出加固

**居中问题二次根因分析：**

前一次修复（`width: fit-content + margin: 0 auto`）只解决了「文本短于容器」的情况。当文本长度超过容器宽度时：
1. `max-width: 100%` 生效，`<p>` 被限制为父容器 100% 宽度
2. `margin: 0 auto` 失效（元素已撑满父容器）
3. `<p>` 自身未设置 `text-align: center`，文本默认左对齐
4. `white-space: nowrap` 导致文本只向右侧溢出
5. `body { overflow-x: hidden }` 裁掉右侧溢出部分
6. 视觉效果：文本看起来左对齐，没有居中

学科竞赛页（42字）文本短，未触发此问题；智能抽取页（52字）和苗绣拼图页文本长，问题暴露。

**最终修复方案：为所有 nowrap 段落补充 `text-align: center`**

在已有 `width: fit-content + max-width: 100% + margin: 0 auto` 基础上，为以下元素新增 `text-align: center`：
- `.hero-desc`（首页描述）
- `.detail-hero-desc`（详情页中文描述）
- `.detail-hero-desc-en`（详情页英文描述）
- `.award-intro-cn`（学科竞赛中文描述）
- `.award-intro-en`（学科竞赛英文描述）

双层保障机制：
- 文本 < 容器宽度：`fit-content` 收缩 + `margin: 0 auto` 居中元素
- 文本 > 容器宽度：`max-width: 100%` 撑开 + `text-align: center` 居中文字，左右对称溢出

**游戏容器溢出加固：**

**右侧导航点滑动动画重构：**

- 新方案：改为「窗口裁剪 + track 滑动」模式
  - HTML：6 个点（对应6个页面）放入 `.page-dots-track` 内层容器，外层 `.page-dots` 作为裁剪窗口
  - 窗口尺寸：`width: 14px; height: 82px; overflow: hidden`（恰好显示5个点：10px×5 + 8px×4 = 82px）
  - `.page-dots-track`：`transition: transform 0.4s cubic-bezier(0.4, 0, 0.2, 1)` 实现平滑滑动
  - JS：`translateY = (2 * DOT_STEP) - (currentSection * DOT_STEP)`，track 先下移2个点距离（让第1个点位于窗口中心），再随当前页上移
  - 效果：页面切换时，整个点列平滑滑动，当前点从相邻位置滑入中心，同时高亮放大，视觉流畅
- 每个点的 class 动态分配：`diff === 0` → active（主色+放大），`|diff| === 1` → near（灰色100%），其余 → far（灰色50%）
- 超出窗口的点被 `overflow: hidden` 裁剪，不可见但不可点击区域自动随 track 移出
